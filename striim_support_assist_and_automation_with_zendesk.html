Support Assist Agent + Automation with Zendesk (Hybrid)

CS handoff view based on what Striim shared and what DevRev validated in POC sessions. This page assumes Zendesk remains the core support platform during the evaluation period.

Model: Hybrid (DevRev + Zendesk) Evaluation Window: 6-12 months Zendesk Renewal: Oct 28, 2026 Blocker: BAA / legal approval

Operating Model Snapshot

  • Current decision: re-sign Zendesk for 12 months while evaluating DevRev AI capabilities in parallel.
  • Primary goal: improve support outcomes using AI (chat, summarization, routing, KB, RCA automation).
  • Customer channels remain Zendesk-first during hybrid; chat widget adds in-app support path.

Support Tiers (Requested)

Tier Role in Hybrid Model Primary System
T1 Initial triage, first response drafts, chatbot-assisted handling, basic workflow execution Zendesk + DevRev assist features
T2 Advanced troubleshooting, summary review, KB enrichment, cross-system coordination Zendesk + DevRev copilots/workflows
T3 Engineering escalation layer (JIRA-linked issues, RCA templates: SCCA/PFA/cloud) JIRA + synced ticket context from DevRev/Zendesk
What We Heard From Striim (Pain Points and Business Drivers)
  • Zendesk AI is viewed as underwhelming; strong interest in AI-first support capabilities.
  • Agents spend 4+ hours reading logs/threads; 200+ message conversations need summarization.
  • Need in-app support via chat widget; trial environment currently blocked for widget activation.
  • Need parity for 20-30 existing Zendesk triggers with role/time-based automation behavior.
  • Need better support-engineering loop through JIRA escalations and standard RCA templates.
  • Need a unified timeline (public + internal comments together) similar to JIRA/Zendesk "All" style.
  • Legal/compliance concern is high: BAA required, HIPAA handling constraints, data boundaries review.
  • Decision pressure tied to Zendesk renewal timeline and budget impact of dual-platform period.
